Git常用命令

发表于2020-08-27|更新于2021-04-03
|阅读量:

准备工作

  • 查看用户名
    git config user.name

  • 查看邮箱号
    git config user.email

  • 全局配置个人信息
    git config --global user.name "Your Name" # 用户名
    git config --global user.email "email@example.com" # 邮箱

  • 查看信息
    git config -l

1.建立一个存储GitHub上代码的本地仓库

git init

2.关联远程仓库

  • 查看远程仓库的信息
    git remote -v

  • 显示某个远程仓库的信息：
    git remote show 标识名(master) 你的远程地址
    例子：

    git remote show https://github.com/Hadoop-cpu/teahouse.git

  • 添加远程仓库的关联
    git remote add 标识名(master) 你的远程地址
    例子：

    git remote add origin https://github.com/Hadoop-cpu/teahouse.git

3.查询日志

git log

4.从远程仓库复制代码

git clone 远程仓库地址
例子：
git clone https://github.com/Hadoop-Memory/teahouse.git

5.查询是否是最新代码

git pull

6.暂存到自己本地仓库

  • 把所有文件添加到缓存中
    git add .

  • 暂存某个文件到本地仓库
    git add 文件名
    例子: 

           git add test.txt

7.本地新增的版本

git commit -m ""
例子:
git commit -m “[order]wangsaisai:新增个人订单查询功能”

8.查询更改内容

git status

9.推到远程仓库（master）

git push origin master

10.分支管理

  • 查看分支
    git branch

  • 创建并切换到dev分支
    git checkout -b dev

  • 切换到主干master
    git checkout master

  • 合并分支dev
    git merge dev

  • 删除远程dev分支
    git push origin --delete dev

11.删除远程仓库文件注意慎用

  • 删除远程 .idea 文件
    git rm -r --cached .idea

    git rm -r --cached **/target/*

    git rm -r --cached **iml

12.分支重命名

1.删除远程分支
git push origin --delete release20180814
2.重命名分支
git branch -m release20180814 dev
3.重命名后的分支提到远程分支
git push origin dev
4.关联一下本地和远程分支
git branch --set-upstream-to=origin/dev
